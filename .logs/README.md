# OpenDash Builder Logs

This directory contains debug logs for the builder widget generation and rendering flow.

## Log Files

### `ai-widget-generation.log`
Tracks all AI widget generation requests from the `/api/builder/ai-widget` endpoint.

**Fields:**
- `[timestamp]` - When the request occurred
- `prompt` - User's natural language prompt
- `widgetId` - ID generated by AI
- `category` - Widget category (charts, stats, etc.)
- `widgetData` - The data structure generated
- Errors and validation messages

**Example entry:**
```
[2026-03-05T20:50:15.123Z] Received prompt: { prompt: "create analytics for sales with line chart" }
[2026-03-05T20:50:15.234Z] AI Response from Groq: { responseText: "{\"widgetId\":\"chart-line\",..." }
[2026-03-05T20:50:15.245Z] SUCCESS: Widget generated: { widgetId: "chart-line", category: "charts" }
```

### `component-render.log`
Tracks component rendering for charts and other widgets in the builder preview.

**Fields:**
- Component name (LineChartBlock, BarChartBlock, etc.)
- Data structure received
- Configuration validation
- Rendering errors

**Example entry:**
```
[LineChartBlock] Rendering with config: { title: "Monthly Sales Trend", xKey: "month", linesCount: 1, dataPoints: 12 }
[BarChartBlock] Missing or empty bars array: undefined
```

## How to Read Logs

### Real-time Monitoring
```bash
tail -f .logs/ai-widget-generation.log
tail -f .logs/component-render.log
```

### Search for Errors
```bash
grep "ERROR\|Missing\|Missing or empty" .logs/*.log
```

### View Latest 50 Lines
```bash
tail -50 .logs/ai-widget-generation.log
```

### Count Successful Generations
```bash
grep "SUCCESS: Widget generated" .logs/ai-widget-generation.log | wc -l
```

## Debugging Chart Generation Issues

### If chart shows "No preview":
1. Check `component-render.log` for `No data provided` or `No line definitions` errors
2. Look in `ai-widget-generation.log` for the prompt that generated the widget
3. Verify the widgetId is either `"chart-line"` or `"chart-bar"`
4. Check that `data` is an array with objects containing the xKey property
5. Check that `lines` (for line charts) or `bars` (for bar charts) is a non-empty array

### Example Debugging Flow:
```bash
# 1. Find when the user added the widget
tail -20 .logs/ai-widget-generation.log

# 2. Look for parsing errors
grep "ERROR\|Exception" .logs/ai-widget-generation.log

# 3. Check rendering logs
grep "LineChartBlock\|BarChartBlock" .logs/component-render.log

# 4. Verify data structure
grep "Parsed widget config" .logs/ai-widget-generation.log | tail -1
```

## Key Troubleshooting Points

### Chart Data Format
✅ **Correct format:**
```json
{
  "widgetId": "chart-line",
  "widgetData": {
    "title": "Sales",
    "xKey": "month",
    "lines": [{"dataKey": "revenue", "label": "Revenue", "color": "#3b82f6"}],
    "data": [
      {"month": "Jan", "revenue": 100},
      {"month": "Feb", "revenue": 120}
    ]
  }
}
```

❌ **Incorrect format (old):**
```json
{
  "widgetId": "line-trend",
  "widgetData": {
    "title": "Sales",
    "points": [100, 120],
    "labels": ["Jan", "Feb"]
  }
}
```

### Component Registration
- `chart-line` → LineChartBlock in component-registry.tsx
- `chart-bar` → BarChartBlock in component-registry.tsx
- Preview functions in presentation/widgets/index.tsx

Both must be present for charts to render!

## Log Rotation
Logs are append-only. To clean up:
```bash
# Archive old logs
mv .logs/ai-widget-generation.log .logs/ai-widget-generation.log.bak

# Restart server to create fresh logs
```
